What clears the highlight
setTimeout(() => testingStore.clearRecentlyChangedTests(), 4000) in the socket store. No component involvement — purely store state.
What drives the CSS fade
transition="background 0.3s, outline-color 0.3s" on <Table.Row>. When recentChangeType becomes null the computed bg changes and Chakra's transition takes over.
Next chat turn resets timer
If the user sends another message, a new REFACTORING_AND_TESTING_UPDATED arrives, overwriting recentlyChangedTests and resetting the 4 s window.

Added (AI-generated new test)
Triggered when a test ID appears in newMissingTests that wasn't in oldMissingTests. Green highlight + NEW badge. Clears after 4 s.
Modified (AI edited existing test)
Triggered when a test ID exists in both old and new but its content differs. Yellow highlight + EDITED badge. Clears after 4 s.
Implementing (task in flight)
Active while implementingTests[test.id] is truthy. Blue tint, spinner in the action button. Persists until task completes.
Priority order matters
AI-highlight states (added, modified) take precedence over blocked, implemented, and default in the bg ternary chain.
