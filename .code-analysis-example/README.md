# Example .code-analysis Folder Structure

This folder demonstrates the expected output structure after running codebase analysis.

## Folder Structure

```
.code-analysis/
├── analysis/
│   └── codebase-analysis.json          # Main codebase analysis output
├── domains/
│   └── {domain-id}/
│       ├── documentation/
│       │   ├── content.md              # Generated markdown documentation
│       │   └── metadata.json           # Metadata: timestamps, tokens, iterations
│       ├── requirements/
│       │   ├── content.json            # Requirements analysis content
│       │   └── metadata.json           # Requirements metadata
│       ├── testing/
│       │   ├── content.json            # Testing analysis content
│       │   ├── metadata.json           # Testing metadata
│       │   └── actions.json            # Recorded implement-test actions
│       ├── bugs-security/
│       │   ├── content.json            # Bugs/security findings
│       │   ├── metadata.json           # Bugs/security metadata
│       │   └── actions.json            # Recorded finding actions
│       └── diagrams/
│           ├── content.json            # Diagram metadata
│           └── metadata.json           # Diagram generation metadata
├── logs/
│   └── {task-id}.log                   # Task execution logs
└── tasks/
    ├── pending/                        # Queued tasks (JSON files)
    ├── completed/                      # Finished tasks (JSON files)
    └── failed/                         # Failed tasks (JSON files)
```

## File Descriptions

### `analysis/codebase-analysis.json`

Main output from analyzing the entire codebase. Contains:

- List of discovered domains
- Project metadata (files, LOC, languages)
- Task execution information
- Token usage statistics

### `domains/{domain-id}/documentation/content.md`

Human-readable Markdown documentation generated by LLM. Includes:

- Business purpose and value
- Core responsibilities
- Key components and their roles
- Architecture and data flow
- Risk areas and security concerns

### `domains/{domain-id}/documentation/metadata.json`

Internal metadata about documentation generation:

- Task ID and timestamps
- Files analyzed
- LLM iterations and token usage
- Conversation log for debugging

### `logs/{task-id}.log`

Detailed execution log for each task showing:

- LLM iterations
- Tool calls (file reads, etc.)
- Progress and completion status
- Any errors or warnings

## Usage

When you run the analyzer on your project, a `.code-analysis/` folder will be created in your project root with this same structure.

Example:

```bash
cd /path/to/your/project
code-analyze
```

This will create:

```
your-project/
├── .code-analysis/
│   ├── analysis/
│   ├── domains/
│   ├── logs/
│   └── tasks/
├── src/
├── package.json
└── ...
```
